I've released [**rflexdid**](https://github.com/austinrobertcraig/rflexdid), an R package implementing the FLEX difference-in-differences estimator for staggered treatment adoption in repeated cross-sections. If you want to skip the background, the [demo vignette](https://github.com/austinrobertcraig/rflexdid/blob/main/vignettes/rflexdid_demo.md) covers all the main options.

---

There has been an [enormous proliferation](https://doi.org/10.1016/j.jeconom.2023.03.008) of new methodologies in difference-in-differences (DiD) design in the past several years. Many of these new designs address the issue that two-way fixed effects (TWFE) estimates may not recover the parameter of interest in cases where the research design deviates from the canonical 2x2 DiD setup.[^1] These issues crop up often in applied work, where treatment timing is often staggered and treatment effects often heterogeneous. Many alternative approaches have been implemented well in R already: [Callaway and Sant'Anna (2021)](https://cran.r-project.org/web/packages/did/index.html), [Sun and Abraham (2021)](https://search.r-project.org/CRAN/refmans/fixest/html/sunab.html), and [Chaisemartin and D’Haultfœuille (2024)](https://asjadnaqvi.github.io/DiD/docs/code_r/07_did_multiplegt_dyn_r/), just to name a few.

[^1]: See, for example, [Goodman-Bacon (2021)](https://doi.org/10.1016/j.jeconom.2021.03.014).

Of course, there is always room for one more DiD R package:

[![](https://imgs.xkcd.com/comics/standards.png)](https://xkcd.com/927/)

I am working on a project which relies on a repeated cross-section (rather than a panel) and came across a working paper by [Deb, Norton, Wooldridge, and Zabel](https://www.nber.org/papers/w33026). They propose FLEX,[^2] a DiD specification robust to staggered treatment adoption and heterogeneous treatment effects in repeated cross-sectional data (though it can also generalize to panel data). Its defining features are the use of a saturated set of group-by-time treatment indicators, the ability to include (potentially time-varying) covariates, and support for both traditional DiD ("lags only") and event study ("leads and lags") designs. The method is estimated by a single OLS regression, so cluster-robust standard errors are valid without the multi-step corrections or bootstrapping required by imputation-based estimators.

[^2]: The name "FLEX" is derived from Flexible Linear model Estimated by OLS with covariates (X).

To the best of my knowledge, no existing R package implements the FLEX specification. `rflexdid` is my R port of the Stata `flexdid` package.[^3] All credit for the underlying methodology and the original Stata implementation belongs to the paper's authors. I wrote this package primarily as a personal research tool, but I am making it public just in case others may find it useful.

[^3]: Deb, P., Norton, E. C., Wooldridge, J. M., & Zabel, J. E. (2025). "A Flexible, Heterogeneous Treatment Effects Difference-in-Differences Estimator for Repeated Cross-Sections." NBER Working Paper No. 33026. Stata package available via SSC: [ideas.repec.org/c/boc/bocode/s459517.html](https://ideas.repec.org/c/boc/bocode/s459517.html).

## Example Usage

To demonstrate, the package ships with a simulated repeated cross-section: 20,000 adults across 40 counties over 10 years, where counties adopt a sugar-sweetened beverage (SSB) tax at staggered times (2013, 2015, or 2017) and the outcome is daily SSB consumption in ounces. Fitting the model with `specification = "lagsandleads"` adds pre-treatment leads for parallel trends testing:

```r
library(rflexdid)
library(fixest)

df <- read.csv(system.file("extdata", "example_data.csv", package = "rflexdid"))

fit <- flexdid(
  ssb_oz ~ 1,
  data          = df,
  tx            = "treated",
  group         = "county",
  time          = "year",
  specification = "lagsandleads",
  vcov          = "cluster",
  cluster       = "county"
)

fit
```

```

##   Observations:  20000

```

The `atet()` function handles aggregation of the estimated treatment effects. The simplest call returns a single pooled treatment effect:

```r
atet(fit, type = "overall")
```

Under staggered adoption with dynamic effects, TWFE is biased: it uses already-treated units as controls for later-treated cohorts, a "forbidden comparison" that contaminates the estimate when effects are still evolving after adoption. Under our DGP, this will attenuate our ATET estimate since larger taxes were imposed on early-adopting counties. The `flexdid` overall ATET recovers an effect closer to the truth.

### Event Study

Passing `type = "byexposure"` to `atet()` produces an event study. Pre-period estimates should be near zero to support the parallel trends assumption; post-period estimates trace how the effect evolves with exposure duration.

```r
ax <- atet(fit, type = "byexposure", test = "pretrends")
ax
plot(ax)
```

![Event study: ATET by years since treatment](/images/posts/2026-05-rflexdid/byexposure-1.png)

The `test = "pretrends"` argument evaluates H₀: all pre-treatment ATETs equal zero. Since the DGP imposes strict parallel pre-trends, we do not reject — consistent with zero pre-treatment differences. Post-period estimates grow steadily more negative with exposure duration.

### By Treatment Cohort

`type = "bycohort"` returns a separate ATET for each treatment cohort — useful for checking whether effect sizes differ across adoption waves.

```r
acoh <- atet(fit, type = "bycohort")
acoh
plot(acoh)
```


### More

`atet()` supports several additional aggregation levels (`bycalendar`, `bygroup`, `byget`). `flexdid()` can handle covariate interactions, subgroup restrictions via `for_expr`, and group-level aggregation weights. The [demo vignette](https://github.com/austinrobertcraig/rflexdid/blob/main/vignettes/rflexdid_demo.md) walks through all of them.
