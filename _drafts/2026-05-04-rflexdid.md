There has been an [enormous proliferation](https://doi.org/10.1016/j.jeconom.2023.03.008) of new methodologies in difference-in-differences (DiD) design in the past several years. We now are well aware that two-way fixed effects (TWFE) estimates may not recover the parameter of interest in cases where the research design deviates from the canonical 2x2 DiD setup - for example, when there is variation in treatment timing and treatment effect heterogeneity.[^1]

[^1]: See, for example, [Goodman-Bacon (2021)](https://doi.org/10.1016/j.jeconom.2021.03.014).

Fortunately, many have been hard at work developing new tools for more varied DiD designs: [Chiu et al. (2026)](https://arxiv.org/pdf/2309.15983) provide a nice overview. Of course, I wasn't quite satisfied with any of the R packages currently available.

![](https://imgs.xkcd.com/comics/standards.png)

Specifically, I am working on a project which relies on a repeated cross-section rather than a panel. The popular [did](https://github.com/bcallaway11/did) package can handle this, but I found it to be somewhat slow and particular about the data. Fortunately, a working paper by [Deb, Norton, Wooldridge, and Zabel](https://www.nber.org/papers/w33026) outlines a method that fits my needs, has an intuitive form, and allows for a wide range of designs.

`rflexdid` is an R port of the Stata `flexdid` package, implementing the estimator developed by Deb, Norton, Wooldridge, and Zabel.[^2] All credit for the underlying methodology and the original Stata implementation belongs to the paper's authors.

[^2]: Deb, P., Norton, E. C., Wooldridge, J. M., & Zabel, J. E. (2025). "A Flexible, Heterogeneous Treatment Effects Difference-in-Differences Estimator for Repeated Cross-Sections." NBER Working Paper No. 33026. Stata package available via SSC: [ideas.repec.org/c/boc/bocode/s459517.html](https://ideas.repec.org/c/boc/bocode/s459517.html).

## Example Usage

To demonstrate, the package ships with a simulated repeated cross-section: 20,000 adults across 40 counties over 10 years, where counties adopt a sugar-sweetened beverage (SSB) tax at staggered times (2013, 2015, or 2017) and the outcome is daily SSB consumption in ounces. Fitting the model with `specification = "lagsandleads"` adds pre-treatment leads for parallel trends testing:

```r
library(rflexdid)
library(fixest)

df <- read.csv(system.file("extdata", "example_data.csv", package = "rflexdid"))

fit <- flexdid(
  ssb_oz ~ 1,
  data          = df,
  tx            = "treated",
  group         = "county",
  time          = "year",
  specification = "lagsandleads",
  vcov          = "cluster",
  cluster       = "county"
)
```

The `atet()` function handles aggregation of the estimated treatment effects. The simplest call returns a single pooled treatment effect:

```r
atet(fit, type = "overall")
```

It's also easy to compare against a standard TWFE specification using `fixest`:

```r
feols(ssb_oz ~ treated | county + year, data = df, cluster = ~ county)
```

Under staggered adoption with dynamic effects, TWFE is biased: it uses already-treated units as controls for later-treated cohorts, a "forbidden comparison" that contaminates the estimate when effects are still evolving after adoption. Under our DGP, this will attenuate our ATET estimate since larger taxes were imposed on early-adopting counties. The `flexdid` overall ATET recovers an effect closer to the truth.

### Event Study

Passing `type = "byexposure"` to `atet()` produces an event study. Pre-period estimates should be near zero to support the parallel trends assumption; post-period estimates trace how the effect evolves with exposure duration.

```r
ax <- atet(fit, type = "byexposure", test = "pretrends")
plot(ax)
```

![Event study: ATET by years since treatment](/images/posts/2026-05-rflexdid/event-study.png)

The `test = "pretrends"` argument evaluates H₀: all pre-treatment ATETs equal zero. Since the DGP imposes strict parallel pre-trends, we do not reject — consistent with zero pre-treatment differences. Post-period estimates grow steadily more negative with exposure duration.

### By Treatment Cohort

`type = "bycohort"` returns a separate ATET for each treatment cohort — useful for checking whether effect sizes differ across adoption waves.

```r
acoh <- atet(fit, type = "bycohort")
plot(acoh)
```

![ATET by treatment cohort](/images/posts/2026-05-rflexdid/by-cohort.png)

The estimates correctly rank 2013 > 2015 > 2017 in magnitude, matching the larger taxes imposed on early-adopter counties in the DGP.

### More

`atet()` supports several additional aggregation levels (`bycalendar`, `bygroup`, `byget`), covariate interactions, subgroup restrictions via `for_expr`, and group-level aggregation weights. The [demo vignette](https://github.com/austinrobertcraig/rflexdid/blob/main/vignettes/rflexdid_demo.md) walks through all of them. The package is primarily a personal research tool that I'm making public in case it's useful to others.
